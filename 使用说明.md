# Light Design 组件库 - 使用说明

## 📖 快速开始

1. 双击打开 `index.html` 即可查看组件库
2. 左侧导航显示所有组件分类
3. 使用顶部搜索框快速查找组件
4. 点击组件名称即可在右侧查看对应的规范图片

## ✨ 新功能

### 🔍 搜索功能
- 左侧顶部的搜索框支持实时搜索组件名称
- 输入关键字即可快速定位目标组件

### 🏷️ 组件元数据
每个组件可以配置：
- **标题**：24px 字体，支持超链接
- **描述**：14px 灰色文字，简要说明组件用途
- **标签**：未开发、模块、公共
- **链接**：点击图标或标题可跳转到详细规范页面

### 📊 分类排序
分类按固定顺序显示：
1. 布局
2. 基础
3. 交互

### 🖼️ 多图支持
支持同一组件的多张图片自动平铺显示。例如：
- `【基础】气泡确认 Popconfirm1.png`
- `【基础】气泡确认 Popconfirm2.png`

这两张图片会在同一页面纵向排列显示。

## 📁 文件命名规范

所有组件图片必须按照以下格式命名：

```
【分类名】组件名称.扩展名
【分类名】组件名称1.扩展名  （多图情况）
【分类名】组件名称2.扩展名  （多图情况）
```

### 示例：
- `【基础】标签Tag.jpg` → 分类：基础，组件名：标签Tag
- `【布局】导航栏.png` → 分类：布局，组件名：导航栏
- `【交互】导入按钮.png` → 分类：交互，组件名：导入按钮
- `【基础】气泡确认 Popconfirm1.png` → 第一张图
- `【基础】气泡确认 Popconfirm2.png` → 第二张图（自动合并）

支持的图片格式：`.png`、`.jpg`、`.jpeg`

## 🔄 添加新组件

### 方法一：使用自动更新工具（推荐）

1. 将新的规范图片放入当前文件夹
2. 确保文件名符合命名规范：`【分类】组件名.扩展名`
3. **双击运行** `更新图片列表.command`
4. 刷新浏览器页面即可看到新组件

### 方法二：使用终端命令

```bash
cd "/Users/sherry/Desktop/light design"
python3 update.py
```

## 🎨 编辑规范（新功能）

### 💡 两种保存模式

#### 模式一：本地模式（默认）
- ✅ 配置保存在浏览器 localStorage
- ✅ 编辑立即生效，无需刷新
- ⚠️ 换浏览器或清除缓存会丢失

#### 模式二：GitHub 模式（推荐用于团队）
- ✅ 配置直接提交到 GitHub 仓库
- ✅ 多人协作，配置同步
- ✅ 永久保存，版本控制
- 需要配置 GitHub 仓库信息

### ✨ 快速编辑

1. 在右侧内容区点击标题旁的 ✏️ **编辑图标**
2. 在弹出的编辑窗口中配置：
   - **标题**：组件名称（自动显示）
   - **描述**：组件功能说明
   - **链接**：详细规范文档的 URL
   - **标签**：点击快捷标签添加（未开发、模块、公共）
   - **图片管理**：标记需要删除的图片
3. 点击 **保存** 按钮 → ✅ **配置立即生效！**

### 🔧 配置 GitHub 模式

如果你通过 GitHub Pages 部署，并希望编辑直接提交到仓库：

**第一步：创建 GitHub Personal Access Token**

1. 访问 [GitHub Token 设置页面](https://github.com/settings/tokens/new)
2. 勾选 `repo` 权限（允许访问仓库）
3. 生成并复制 Token（格式：`ghp_xxxxxxxxxxxx`）

**第二步：配置仓库信息**

1. 点击顶部导航栏的 **"GitHub 设置"** 按钮
2. 填写以下信息：
   - **GitHub 用户名**：你的 GitHub 用户名
   - **仓库名称**：例如 `light-design`
   - **分支名称**：通常是 `main` 或 `master`
   - **文件路径**：`index.html`（或你的实际路径）
   - **Personal Access Token**：粘贴刚才创建的 Token
3. 点击 **"测试连接"** 验证配置
4. 勾选 **"启用 GitHub 直接保存模式"**
5. 点击 **"保存设置"**

**第三步：开始使用**

配置完成后，顶部会显示 🟢 **"GitHub 模式"** 指示器。

- 编辑并保存时，配置会直接提交到 GitHub
- 提交后页面自动刷新，加载最新内容
- 其他人访问也会看到更新后的配置

**备份配置**
- 点击编辑窗口中的"备份配置"按钮
- 复制生成的代码用于备份或分享

**重置配置**
- 点击顶部导航栏的"重置配置"按钮

**切换模式**

### 标签类型：
- **未开发**：🟡 橙色标签，表示组件尚未开发
- **模块**：🔵 蓝色标签，表示模块级组件
- **公共**：🟢 绿色标签，表示公共组件

### 图片管理：
- **标记删除**：在编辑器中标记需要删除的图片
- **手动删除**：从文件夹中删除对应的图片文件
- **更新列表**：运行"更新图片列表.command"刷新显示

### 💡 技术说明

**本地模式**
- 配置保存在浏览器的 localStorage 中
- 换浏览器或清除缓存会丢失（建议定期备份）
- 适合个人使用或临时编辑

**GitHub 模式**
- 使用 GitHub API 直接更新仓库文件
- Token 和配置保存在浏览器 localStorage
- 提交信息：`更新组件规范配置 - [时间]`
- 适合团队协作和永久保存
- ⚠️ Token 仅存在本地浏览器，不会上传

**安全提示**
- Personal Access Token 具有仓库访问权限
- 建议创建专用 Token，仅授予必要权限
- 不要在公共场合分享 Token
- 定期更新和轮换 Token

## ⚠️ 注意事项

1. 图片文件名必须包含 `【分类】` 标签，否则不会显示
2. 分类名使用中文全角括号 `【】`
3. 添加新图片后必须运行更新工具
4. 分类顺序固定为：布局 → 基础 → 交互
5. 带编号的图片会自动合并显示（如 name1.png, name2.png）
