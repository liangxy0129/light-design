# Light Design 组件库 - 使用说明

## 📖 快速开始

1. 双击打开 `index.html` 即可查看组件库
2. 左侧导航显示所有组件分类
3. 使用顶部搜索框快速查找组件
4. 点击组件名称即可在右侧查看对应的规范图片

## ✨ 新功能

### 🔍 搜索功能
- 左侧顶部的搜索框支持实时搜索组件名称
- 输入关键字即可快速定位目标组件

### 🏷️ 组件元数据
每个组件可以配置：
- **标题**：24px 字体，支持超链接
- **描述**：14px 灰色文字，简要说明组件用途
- **标签**：未开发、模块、公共
- **链接**：点击图标或标题可跳转到详细规范页面

### 📊 分类排序
分类基于 `standard-pic` 文件夹下的子文件夹自动生成，按文件夹名称开头的数字排序：
1. `1基础` → 第 1 组
2. `2 布局与导航` → 第 2 组
3. `3表单与数据` → 第 3 组
4. `4浮层与反馈` → 第 4 组

### 🖼️ 多图支持
支持同一组件的多张图片自动平铺显示。例如：
- `导入规范-1.png`
- `导入规范-2.png`

这两张图片会在同一页面纵向排列显示。

## 📁 文件组织规范

### 文件夹结构

所有组件图片必须放置在 `standard-pic` 文件夹下的分类子文件夹中：

```
standard-pic/
├── 1基础/
│   └── 文本域 TextArea.png
├── 2 布局与导航/
│   ├── 导航栏.png
│   └── 标签页 Tabs.png
├── 3表单与数据/
│   ├── 列表规范-1.png
│   └── 列表规范-2.png
└── 4浮层与反馈/
    └── 气泡确认 Popconfirm.png
```

### 文件夹命名规则

- **格式**：`数字 + 分类名称`
- **数字**：决定分类的排序权重（越小越靠前）
- **分类名称**：显示在左侧导航的分组标题

### 示例：
- `1基础` → 排序第 1，显示为"基础"
- `2 布局与导航` → 排序第 2，显示为"布局与导航"（数字后可带空格）
- `3表单与数据` → 排序第 3，显示为"表单与数据"
- `4浮层与反馈` → 排序第 4，显示为"浮层与反馈"

### 文件命名规则

- 文件名直接使用组件名称，无需添加 `【分类】` 前缀
- 多图情况使用 `-1`、`-2` 或空格 `1`、`2` 后缀
- 支持的图片格式：`.png`、`.jpg`、`.jpeg`

### 命名示例：
- `文本域 TextArea.png` → 组件名：文本域 TextArea
- `列表规范-1.png` → 组件名：列表规范（第 1 张）
- `列表规范-2.png` → 第二张图（自动合并显示）
- `导航栏.png` → 组件名：导航栏

## 🔄 添加新组件

### 方法一：使用自动更新工具（推荐）

1. 将新的规范图片放入对应的分类文件夹中
   - 例如：`standard-pic/1基础/新组件.png`
2. 如果需要新建分类，创建新文件夹并以数字开头
   - 例如：`standard-pic/5新分类/`
3. **双击运行** `更新图片列表.command`
4. 刷新浏览器页面即可看到新组件

### 方法二：使用终端命令

```bash
cd "/Users/sherry/Desktop/light design"
python3 update.py
```

## 🎨 编辑规范（新功能）

### 💡 两种保存模式

#### 模式一：本地模式（默认）
- ✅ 配置保存在浏览器 localStorage
- ✅ 编辑立即生效，无需刷新
- ⚠️ 换浏览器或清除缓存会丢失

#### 模式二：GitHub 模式（推荐用于团队）
- ✅ 配置直接提交到 GitHub 仓库
- ✅ 多人协作，配置同步
- ✅ 永久保存，版本控制
- 需要配置 GitHub 仓库信息

---

### ✨ 快速编辑

1. 在右侧内容区点击标题旁的 ✏️ **编辑图标**
2. 在弹出的编辑窗口中配置：
   - **标题**：组件名称（自动显示）
   - **描述**：组件功能说明
   - **链接**：详细规范文档的 URL
   - **标签**：点击快捷标签添加（未开发、模块、公共）
   - **图片管理**：标记需要删除的图片
3. 点击 **保存** 按钮 → ✅ **配置立即生效！**

### 🔧 配置 GitHub 模式

如果你通过 GitHub Pages 部署，并希望编辑直接提交到仓库：

**第一步：创建 GitHub Personal Access Token**

1. 访问 [GitHub Token 设置页面](https://github.com/settings/tokens/new)
2. 勾选 `repo` 权限（允许访问仓库）
3. 生成并复制 Token（格式：`ghp_xxxxxxxxxxxx`）

**第二步：配置仓库信息**

1. 点击顶部导航栏的 **"GitHub 设置"** 按钮
2. 填写以下信息：
   - **GitHub 用户名**：你的 GitHub 用户名
   - **仓库名称**：例如 `light-design`
   - **分支名称**：通常是 `main` 或 `master`
   - **文件路径**：`index.html`（或你的实际路径）
   - **Personal Access Token**：粘贴刚才创建的 Token
3. 点击 **"测试连接"** 验证配置
4. 勾选 **"启用 GitHub 直接保存模式"**
5. 点击 **"保存设置"**

**第三步：开始使用**

配置完成后，顶部会显示 🟢 **"GitHub 模式"** 指示器。

- 编辑并保存时，配置会直接提交到 GitHub
- 提交后页面自动刷新，加载最新内容
- 其他人访问也会看到更新后的配置

### 🔄 管理配置

**备份配置**
- 点击编辑窗口中的"备份配置"按钮
- 复制生成的代码用于备份或分享

**重置配置**

**切换模式**
- 随时可以在 GitHub 设置中启用/禁用 GitHub 模式
- 本地模式和 GitHub 模式可以灵活切换

### 标签类型：
- **未开发**：🟡 橙色标签，表示组件尚未开发
- **模块**：🔵 蓝色标签，表示模块级组件
- **公共**：🟢 绿色标签，表示公共组件

### 图片管理：
- **标记删除**：在编辑器中标记需要删除的图片
- **手动删除**：从文件夹中删除对应的图片文件
- **更新列表**：运行"更新图片列表.command"刷新显示

### 💡 技术说明

**本地模式**
- 配置保存在浏览器的 localStorage 中
- 换浏览器或清除缓存会丢失（建议定期备份）
- 适合个人使用或临时编辑

**GitHub 模式**
- 使用 GitHub API 直接更新仓库文件
- Token 和配置保存在浏览器 localStorage
- 提交信息：`更新组件规范配置 - [时间]`
- 适合团队协作和永久保存
- ⚠️ Token 仅存在本地浏览器，不会上传

**安全提示**
- Personal Access Token 具有仓库访问权限
- 建议创建专用 Token，仅授予必要权限
- 不要在公共场合分享 Token
- 定期更新和轮换 Token

## ⚠️ 注意事项

1. 所有图片必须放置在 `standard-pic` 文件夹下的子文件夹中
2. 文件夹名必须以数字开头，数字决定分类排序
3. 添加新图片或新文件夹后必须运行更新工具
4. 分类顺序由文件夹名的数字前缀自动决定
5. 带编号的图片会自动合并显示（如 `name-1.png`, `name-2.png`）
6. 不要将图片直接放在 `standard-pic` 根目录下
